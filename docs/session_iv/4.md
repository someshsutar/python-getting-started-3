# Debugging with `pdb` in Python

`pdb` (Python Debugger) is Python's built-in interactive debugger. It allows you to **pause program execution**, inspect variables, execute code line by line, modify variable values, and identify bugs efficiently.

Unlike `print()` debugging, `pdb` gives you complete control over the execution flow.

---

# What is `pdb`?

`pdb` stands for **Python Debugger**.

It helps you:

* Pause execution at a specific line (breakpoint)
* Inspect variables
* Step through code line by line
* Execute arbitrary Python statements
* Modify variables during execution
* Analyze the call stack

Since `pdb` is part of Python's standard library, **no installation is required**.

---

# Why Use `pdb`?

Suppose a program produces incorrect output.

Instead of adding multiple `print()` statements like:

```python
print(a)
print(b)
print(result)
```

You can stop execution exactly where needed and inspect all variables interactively.

Advantages:

* Faster debugging
* No need to modify code repeatedly
* Easy inspection of program state
* Supports stepping through execution

---

# Using `pdb.set_trace()`

The simplest way to start debugging is to insert a breakpoint using `pdb.set_trace()`.

Example:

```python
import pdb

a = 10
b = 20

pdb.set_trace()     # Breakpoint

c = a + b

print(c)
```

Run the program:

```bash
python program.py
```

Output:

```text
> program.py(8)<module>()
-> c = a + b
(Pdb)
```

The `(Pdb)` prompt indicates that the debugger is active.

---

# Example 1: Inspect Variables

```python
import pdb

x = 100
y = 50

pdb.set_trace()

z = x + y

print(z)
```

Debugger session:

```text
(Pdb) p x
100

(Pdb) p y
50

(Pdb) p x + y
150
```

The `p` command prints variables or expressions.

---

# Example 2: Step Through Code

```python
import pdb

a = 5
b = 4

pdb.set_trace()

c = a * b
d = c + 10

print(d)
```

Debugger:

```text
(Pdb) n
```

Moves to:

```python
d = c + 10
```

Again:

```text
(Pdb) n
```

Moves to:

```python
print(d)
```

# Common `pdb` Commands

| Command       | Description                              |
| ------------- | ---------------------------------------- |
| `h`           | Help                                     |
| `l`           | List source code                         |
| `n`           | Execute next line (Step Over)            |
| `s`           | Step into a function                     |
| `r`           | Continue until current function returns  |
| `c`           | Continue execution until next breakpoint |
| `q`           | Quit debugger                            |
| `p variable`  | Print variable or expression             |
| `pp variable` | Pretty-print complex objects             |
| `w`           | Show stack trace                         |
| `u`           | Move up one stack frame                  |
| `d`           | Move down one stack frame                |
| `args`        | Show function arguments                  |
| `where`       | Display current call stack               |
| `break`       | Set a breakpoint                         |
| `clear`       | Remove breakpoints                       |

# Step Into (`s`)

Consider:

```python
import pdb

def multiply(a, b):
    return a * b

x = 5
y = 8

pdb.set_trace()

result = multiply(x, y)

print(result)
```

Debugger:

```text
(Pdb) s
```

Now execution moves inside:

```python
def multiply(a, b):
```

This is useful when debugging function logic.

---

# Step Over (`n`)

Instead of entering the function:

```text
(Pdb) n
```

The function executes completely, and execution moves to the next line.

---

# Continue (`c`)

If everything looks correct:

```text
(Pdb) c
```

The program resumes until:

* another breakpoint
* the end of the program

---

# Quit (`q`)

```text
(Pdb) q
```

Terminates debugging immediately.

---

# Listing Source Code

```text
(Pdb) l
```

Example:

```text

 2
 3 a = 10
 4 b = 20
 5
 6 pdb.set_trace()
 7
 8 c = a + b

```

# Pretty Printing Objects

Instead of:

```text
(Pdb) p student
```

Use:

```text
(Pdb) pp student
```

Example:

```python
student = {
    "name": "John",
    "marks": [80, 90, 95]
}
```

Output:

```text
{'marks': [80, 90, 95],
 'name': 'John'}
```

# Executing Python Statements

Inside the debugger:

```text
(Pdb) x = 200
```

Now:

```text
(Pdb) p x
200
```

You can even execute expressions:

```text
(Pdb) x + 100
300
```

# Debugging Functions

```python
import pdb

def calculate_total(price, quantity):

    pdb.set_trace()

    total = price * quantity

    discount = total * 0.10

    return total - discount

print(calculate_total(500, 4))
```

Debugger session:

```text
(Pdb) p price
500

(Pdb) p quantity
4

(Pdb) n

(Pdb) p total
2000

(Pdb) n

(Pdb) p discount
200
```

# Debugging Exceptions

```python
import pdb

def divide(a, b):
    return a / b

pdb.set_trace()

print(divide(10, 0))
```

You can inspect variables before the exception occurs.

# Running a Program with `pdb`

Instead of modifying the source code:

```bash
python -m pdb program.py
```

Example:

```bash
python -m pdb calculator.py
```

Execution starts at the first line.

Useful commands:

```text
(Pdb) b 15
```

Sets breakpoint at line 15.

```text
(Pdb) c
```

Continues until line 15.

---

# Breakpoints

Example:

```python
x = 10
y = 20
z = x + y

print(z)
```

Run:

```bash
python -m pdb program.py
```

Debugger:

```text
(Pdb) b 3
Breakpoint 1 at line 3
```

Continue:

```text
(Pdb) c
```

Execution stops at line 3.

# Real-World Example: Online Shopping Discount

```python
import pdb

def calculate_bill(price, quantity, discount):

    pdb.set_trace()

    total = price * quantity

    final = total - discount

    return final

print(calculate_bill(1000, 2, 100))
```

Debugger walkthrough:

```text
(Pdb) p price
1000

(Pdb) p quantity
2

(Pdb) n

(Pdb) p total
2000

(Pdb) n

(Pdb) p final
1900

(Pdb) c
1900
```

This allows you to verify every calculation before the function returns.

# `print()` Debugging vs `pdb`

| Feature                       | `print()` | `pdb`                                    |
| ----------------------------- | --------- | ---------------------------------------- |
| Inspect variables             | Yes       | Yes                                      |
| Step through code             | No        | Yes                                      |
| Pause execution               | No        | Yes                                      |
| Modify variables              | No        | Yes                                      |
| Call stack inspection         | No        | Yes                                      |
| Execute Python code           | No        | Yes                                      |
| Remove debug statements later | Required  | Usually not (when using `python -m pdb`) |

---

# Best Practices

* Use `pdb` for complex debugging instead of scattering `print()` statements.
* Place breakpoints close to where you suspect the bug.
* Inspect variables before changing code.
* Use `n` (next) to understand program flow line by line.
* Use `s` (step) when you need to inspect function internals.
* Remove `pdb.set_trace()` statements before committing code.
* In Python 3.7+, you can use the built-in `breakpoint()` function instead of `pdb.set_trace()`. It invokes the configured debugger and is generally preferred.

Example:

```python
def calculate_sum(a, b):
    breakpoint()   # Starts the debugger
    return a + b
```
