# Logging for Monitoring in Python

Logging is the process of recording events, messages, warnings, and errors that occur while a program is running. Unlike `print()` statements, logging provides configurable, structured, and persistent records that help with **monitoring**, **debugging**, **troubleshooting**, and **auditing** applications.

The built-in **`logging`** module is the standard way to implement logging in Python.

---

# Why Use Logging?

Logging helps you:

* Monitor application health
* Debug issues in production
* Record warnings and errors
* Audit important events
* Analyze application performance
* Track user actions (without logging sensitive data)

For example, an e-commerce application might log:

* User login attempts
* Order placement
* Payment success or failure
* Database connection issues

---

# Logging vs `print()`

| `print()`                         | Logging                             |
| --------------------------------- | ----------------------------------- |
| Temporary debugging               | Long-term monitoring                |
| Output only to console            | Console, files, or external systems |
| No log levels                     | Supports multiple log levels        |
| Difficult to manage               | Configurable and structured         |
| Cannot easily disable by severity | Filter by severity level            |

---

# Logging Levels

Python defines five commonly used logging levels:

| Level      | Numeric Value | Purpose                                     |
| ---------- | ------------: | ------------------------------------------- |
| `DEBUG`    |            10 | Detailed diagnostic information             |
| `INFO`     |            20 | General application events                  |
| `WARNING`  |            30 | Unexpected situations that are not fatal    |
| `ERROR`    |            40 | Errors that prevent an operation            |
| `CRITICAL` |            50 | Severe errors that may stop the application |

Example:

```text id="5rj9nq"
DEBUG
   ↓
INFO
   ↓
WARNING
   ↓
ERROR
   ↓
CRITICAL
```

# Basic Logging Example

```python id="2okc85"
import logging

logging.basicConfig(level=logging.INFO)

logging.info("Application started")
logging.warning("Disk space is low")
logging.error("Database connection failed")
```

### Output

```text id="aotkiv"

WARNING:root:Disk space is low
ERROR:root:Database connection failed
```

# Logging to a File

Instead of printing logs to the console, store them in a file.

```python id="jyc5qk"
import logging

logging.basicConfig(
    filename="application.log",
    level=logging.INFO
)

logging.info("Program started")
logging.warning("Memory usage is high")
logging.error("Unable to connect to server")
```

Generated file (`application.log`):

```text id="oj7c6m"
INFO:root:Program started
WARNING:root:Memory usage is high
ERROR:root:Unable to connect to server
```

# Custom Log Format

You can customize the log message format.

```python id="6ow4al"
import logging

logging.basicConfig(
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s"
)

logging.info("Application started")
```

### Example Output

```text id="8kvyit"
2026-07-19 10:15:20,500 INFO Application started
```

Useful formatting fields:

| Placeholder     | Meaning          |
| --------------- | ---------------- |
| `%(asctime)s`   | Date and time    |
| `%(levelname)s` | Log level        |
| `%(message)s`   | Log message      |
| `%(filename)s`  | Source file name |
| `%(funcName)s`  | Function name    |
| `%(lineno)d`    | Line number      |
| `%(name)s`      | Logger name      |

# Creating a Logger

Instead of using the root logger, create a named logger.

```python id="cjlwmv"
import logging

logger = logging.getLogger("OrderService")

logging.basicConfig(level=logging.INFO)

logger.info("Order created successfully")
```

Output:

```text id="9kk55v"
INFO:OrderService:Order created successfully
```

Named loggers make it easier to identify which module produced the log.

# Logging Exceptions

Instead of manually printing exceptions:

```python id="r7gpxh"
try:
    10 / 0
except Exception as e:
    logging.error(e)
```

A better approach:

```python id="jjv9g0"
import logging

logging.basicConfig(level=logging.ERROR)

try:
    10 / 0
except Exception:
    logging.exception("An exception occurred")
```

### Output

```text id="z8lk4w"
ERROR:root:An exception occurred
Traceback (most recent call last):
...
ZeroDivisionError: division by zero
```

`logging.exception()` automatically includes the full stack trace.

# Logging Inside Functions

```python id="3q8wx8"
import logging

logging.basicConfig(level=logging.INFO)

def calculate_salary(hours, rate):

    logging.info("Calculating salary")

    salary = hours * rate

    logging.info(f"Salary = {salary}")

    return salary

calculate_salary(40, 500)
```

Output:

```text id="frd6xh"

INFO:root:Salary = 20000
```

# Rotating Log Files

Large log files can consume significant disk space. Use rotating handlers.

```python id="rv8fuv"
import logging
from logging.handlers import RotatingFileHandler

handler = RotatingFileHandler(
    "app.log",
    maxBytes=1024,
    backupCount=3
)

logging.basicConfig(
    handlers=[handler],
    level=logging.INFO
)

logging.info("Application started")
```

When `app.log` reaches 1 KB, it is rotated:

```text id="ec5gph"
app.log
app.log.1
app.log.2
app.log.3
```

---

# Timed Log Rotation

Create a new log file at regular intervals.

```python id="5bsqyr"
from logging.handlers import TimedRotatingFileHandler
import logging

handler = TimedRotatingFileHandler(
    "app.log",
    when="midnight",
    backupCount=7
)

logging.basicConfig(
    handlers=[handler],
    level=logging.INFO
)
```

This creates a new log file every midnight and keeps the last 7 log files.

# Best Practices

* Use `DEBUG` for detailed diagnostic information.
* Use `INFO` for normal application events.
* Use `WARNING` for recoverable problems.
* Use `ERROR` when an operation fails.
* Use `CRITICAL` for severe failures requiring immediate attention.
* Create a separate logger for each module with `logging.getLogger(__name__)`.
* Use `logging.exception()` inside `except` blocks to capture stack traces.
* Rotate log files using `RotatingFileHandler` or `TimedRotatingFileHandler`.
* Avoid logging passwords, API keys, tokens, or other sensitive information.
* Configure logging once, typically in your application's entry point.

# Real-World Mini Project: Banking Transaction Logger

```python id="f3h1lb"
import logging

logging.basicConfig(
    filename="bank.log",
    level=logging.INFO,
    format="%(asctime)s %(levelname)s %(message)s"
)

def withdraw(balance, amount):

    logging.info(f"Withdrawal requested: {amount}")

    if amount > balance:
        logging.error("Insufficient balance")
        return balance

    balance -= amount

    logging.info(f"Withdrawal successful. Remaining balance: {balance}")

    return balance

balance = 5000

balance = withdraw(balance, 1000)
balance = withdraw(balance, 7000)
```

Generated log file:

```text id="9dylo7"
INFO Withdrawal requested: 1000
INFO Withdrawal successful. Remaining balance: 4000
INFO Withdrawal requested: 7000
ERROR Insufficient balance
```

This log can be used to monitor transaction activity and diagnose issues without interrupting the user experience.
