# Assertions in Python

An **assertion** is a debugging aid that checks whether a condition is true. If the condition is **True**, the program continues execution. If it is **False**, Python raises an **`AssertionError`**.

Assertions are commonly used to verify assumptions in code, detect programming errors early, and write unit tests.

---

# Syntax of `assert`

```python
assert condition
```

or

```python
assert condition, "Error message"
```

If the condition evaluates to `False`, Python raises an `AssertionError`.

---

# How Assertions Work

```text
Evaluate Condition
        │
        ▼
 Is condition True?
    │          │
   Yes        No
    │          │
Continue   Raise AssertionError
```

---

# Example 1: Basic Assertion

```python
age = 20

assert age >= 18

print("Eligible to vote")
```

### Output

```text
Eligible to vote
```

Since the condition is `True`, the program continues.

---

# Example 2: Failed Assertion

```python
age = 15

assert age >= 18

print("Eligible to vote")
```

### Output

```text
Traceback (most recent call last):
...
AssertionError
```

The program stops because the condition is `False`.

---

# Example 3: Assertion with a Message

```python
age = 15

assert age >= 18, "Person must be at least 18 years old."

print("Eligible")
```

### Output

```text
Traceback (most recent call last):
...
AssertionError: Person must be at least 18 years old.
```

Providing a message makes debugging easier.

---

# Common Uses of Assertions

## 1. Validate Function Inputs

```python
def calculate_square_root(number):
    assert number >= 0, "Number cannot be negative"
    return number ** 0.5

print(calculate_square_root(25))
```

### Output

```text
5.0
```

---

## 2. Verify Internal Assumptions

```python
numbers = [10, 20, 30]

assert len(numbers) == 3

print("List size is correct")
```

---

## 3. Detect Logical Errors

```python
result = 10 + 20

assert result == 30

print("Calculation is correct")
```

---

# Assertions Inside Functions

```python
def divide(a, b):
    assert b != 0, "Division by zero is not allowed"
    return a / b

print(divide(10, 2))
```

### Output

```text
5.0
```

If called as:

```python
divide(10, 0)
```

Output:

```text
AssertionError: Division by zero is not allowed
```

---

# Assertions in Loops

```python
marks = [80, 75, 90, 85]

for mark in marks:
    assert mark >= 0

print("All marks are valid")
```

---

# Assertions in Object-Oriented Programming

```python
class BankAccount:

    def __init__(self, balance):
        assert balance >= 0, "Balance cannot be negative"
        self.balance = balance

account = BankAccount(5000)

print(account.balance)
```

# Assertions in Unit Testing

Assertions are widely used in testing frameworks to verify expected outcomes.

## Using `unittest`

```python
import unittest

class TestMath(unittest.TestCase):

    def test_addition(self):
        self.assertEqual(2 + 3, 5)

if __name__ == "__main__":
    unittest.main()
```

## Using `pytest`

```python
def add(a, b):
    return a + b

def test_add():
    assert add(2, 3) == 5
```

Here, `pytest` uses Python's built-in `assert` statement, while `unittest` provides specialized assertion methods.

---

# AssertionError Exception

Assertions raise an `AssertionError`.

```python
try:
    assert 5 > 10, "Invalid comparison"
except AssertionError as e:
    print(e)
```

### Output

```text
Invalid comparison
```

---

# Assertions Can Be Disabled

Python can disable assertions using the optimization (`-O`) flag.

```bash
python -O program.py
```

When run with `-O`:

* All `assert` statements are ignored.
* No `AssertionError` is raised.
* The program executes as if assertions were not present.

Because of this, **assertions should not be used to enforce essential runtime validations**.

---

# Assertions vs Exception Handling

| Assertion                          | Exception Handling                                                      |
| ---------------------------------- | ----------------------------------------------------------------------- |
| Used to catch programming mistakes | Used to handle expected runtime errors                                  |
| Raises `AssertionError`            | Raises or handles specific exceptions (`ValueError`, `TypeError`, etc.) |
| Can be disabled with `-O`          | Always active                                                           |
| Mainly for debugging and testing   | For production error handling                                           |

### Example

**Using Assertion**

```python
assert x > 0, "x must be positive"
```

**Using Exception**

```python
if x <= 0:
    raise ValueError("x must be positive")
```

For user input or business rules, prefer exceptions over assertions.

# Real-World Example: Online Shopping

```python
class ShoppingCart:

    def __init__(self):
        self.items = []

    def add_item(self, item, quantity):
        assert quantity > 0, "Quantity must be greater than zero"
        self.items.append((item, quantity))

cart = ShoppingCart()

cart.add_item("Laptop", 2)

print(cart.items)
```

### Output

```text
[('Laptop', 2)]
```

If:

```python
cart.add_item("Laptop", -2)
```

Output:

```text
AssertionError: Quantity must be greater than zero
```

---

# Best Practices

* Use assertions to verify internal assumptions and detect programming errors.
* Include clear, descriptive assertion messages.
* Do not use assertions for validating external input (user input, API requests, files, etc.).
* Use exceptions (`ValueError`, `TypeError`, etc.) for production input validation.
* Keep assertion conditions simple and easy to understand.
* Remember that assertions may be disabled with Python's `-O` optimization flag.

---
