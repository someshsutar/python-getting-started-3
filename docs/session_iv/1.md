# Testing & Debugging in Python

Testing and debugging are essential practices in Python development. **Testing** ensures your code behaves as expected, while **debugging** helps identify and fix errors when your program does not work correctly.

---

# 1. What is Testing?

Testing is the process of verifying that a program produces the expected output for different inputs.

## Objectives of Testing

* Detect bugs early
* Ensure correctness of code
* Prevent future regressions
* Improve software reliability
* Make code easier to maintain

---

# 2. Types of Testing

## A. Unit Testing

Tests individual functions or methods independently.

Example:

```python
def add(a, b):
    return a + b
```

Test:

```python
import unittest

def add(a, b):
    return a + b

class TestMath(unittest.TestCase):

    def test_add(self):
        self.assertEqual(add(2, 3), 5)

if __name__ == "__main__":
    unittest.main()
```

Output

```
.
----------------------------------------------------------------------
Ran 1 test

OK
```

---

## B. Integration Testing

Tests interaction between multiple modules.

Example

```
Database + API + Business Logic
```

Instead of testing each separately, integration testing verifies they work together.

---

## C. Functional Testing

Tests complete application functionality from the user's perspective.

Example

```
Login
↓

Enter username/password

↓

Verify dashboard opens
```

---

## D. System Testing

Tests the entire software system.

Example

```
E-commerce Application

User Login
↓

Search Product
↓

Add to Cart
↓

Payment
↓

Order Confirmation
```

---

## E. Acceptance Testing

Performed by clients/end users to verify business requirements.

Example

Customer verifies:

* Reports generated correctly
* Login works
* Data exported successfully

# 4. unittest Module

Python's built-in testing library.

Basic Structure

```python
import unittest

class TestClass(unittest.TestCase):

    def test_case(self):
        self.assertEqual(2 + 3, 5)

if __name__ == "__main__":
    unittest.main()
```

---

## Common Assertions

```python
assertEqual(a, b)
assertNotEqual(a, b)
assertTrue(x)
assertFalse(x)
assertIsNone(x)
assertIsNotNone(x)
assertIn(a, b)
assertNotIn(a, b)
assertGreater(a, b)
assertLess(a, b)
```

Example

```python
import unittest

class TestAssertions(unittest.TestCase):

    def test_values(self):
        self.assertEqual(10, 10)
        self.assertTrue(5 > 2)
        self.assertIn("a", "apple")

if __name__ == "__main__":
    unittest.main()
```

# 6. Test-Driven Development (TDD)

TDD follows:

```
Write Test
↓

Run Test (Fails)

↓

Write Code

↓

Run Test (Passes)

↓

Refactor
```

Example

Step 1

```python
def test_square():
    assert square(5) == 25
```

Step 2

```python
def square(x):
    return x * x
```

# 12. Best Practices for Testing

* Write small, independent test cases.
* Test both valid and invalid inputs.
* Use descriptive test names.
* Automate tests using `unittest` or `pytest`.
* Aim for good code coverage, but prioritize meaningful tests.
* Run tests whenever code changes.
* Keep tests deterministic and isolated.

---

# 13. Best Practices for Debugging

* Read the complete error traceback carefully.
* Reproduce the issue consistently before fixing it.
* Use a debugger instead of relying only on `print()` statements.
* Add logging for complex applications.
* Fix the root cause, not just the symptoms.
* Write a test for every bug you fix to prevent regressions.
* Simplify the problem by isolating the failing code.

# Real-World Example: Testing an Online Banking Application

Suppose you have a function that transfers money between accounts.

```python
def transfer(balance, amount):
    if amount <= 0:
        raise ValueError("Amount must be positive")
    if amount > balance:
        raise ValueError("Insufficient balance")
    return balance - amount
```

### Unit Tests

```python
import unittest

class TestTransfer(unittest.TestCase):

    def test_valid_transfer(self):
        self.assertEqual(transfer(1000, 200), 800)

    def test_zero_amount(self):
        with self.assertRaises(ValueError):
            transfer(1000, 0)

    def test_negative_amount(self):
        with self.assertRaises(ValueError):
            transfer(1000, -100)

    def test_insufficient_balance(self):
        with self.assertRaises(ValueError):
            transfer(500, 1000)

if __name__ == "__main__":
    unittest.main()
```

This example demonstrates:

* **Unit Testing** using `unittest`
* **Assertion methods** (`assertEqual`, `assertRaises`)
* **Exception testing** for invalid scenarios
* **Positive and negative test cases**
