# Generators in Python

A **generator** is a special type of function that **produces values one at a time, on demand**, instead of computing and storing all values in memory at once.

Generators use the **`yield`** keyword instead of `return`.

They are extremely useful when working with **large datasets**, **file processing**, **data streams**, or **infinite sequences** because they are memory-efficient.

---

# Why do we need generators?

Suppose you want numbers from 1 to 1,000,000.

### Using a list

```python
def numbers():
    result = []
    for i in range(1, 1000001):
        result.append(i)
    return result

nums = numbers()
```

This creates a list containing **all one million numbers** in memory.

### Using a generator

```python
def numbers():
    for i in range(1, 1000001):
        yield i

nums = numbers()
```

No list is created.

The generator produces one number at a time, only when requested.

---

# `return` vs `yield`

### Using `return`

```python
def greet():
    return "Hello"
```

```python
print(greet())
```

Output

```text
Hello
```

After `return`, the function ends.

---

### Using `yield`

```python
def greet():
    yield "Hello"
```

```python
g = greet()

print(next(g))
```

Output

```text
Hello
```

The function **pauses** at `yield` instead of ending.

---

# First Generator Example

```python
def count():
    yield 1
    yield 2
    yield 3

g = count()

print(next(g))
print(next(g))
print(next(g))
```

Output

```text
1
2
3
```

After the third value:

```python
print(next(g))
```

Output

```text
StopIteration
```

The generator has no more values to produce.

# How a Generator Works

```python
def numbers():
    print("Start")
    yield 1

    print("Middle")
    yield 2

    print("End")
    yield 3
```

Execution:

```python
g = numbers()
```

Nothing runs yet.

---

First call:

```python
next(g)
```

Output

```text
Start
1
```

Execution pauses at the first `yield`.

---

Second call:

```python
next(g)
```

Output

```text
Middle
2
```

The function resumes exactly where it paused.

---

Third call:

```python
next(g)
```

Output

```text
End
3
```

---

Visualization

```text
Generator Created
        │
        ▼
next()
        │
        ▼
Runs until yield
        │
        ▼
Pauses
        │
        ▼
next()
        │
        ▼
Resumes from previous yield
```

# Iterating over a Generator

Instead of calling `next()` manually:

```python
def count():
    yield 1
    yield 2
    yield 3

for number in count():
    print(number)
```

Output

```text
1
2
3
```

The `for` loop automatically calls `next()` until `StopIteration`.

---

# Generator with a Loop

```python
def squares(n):

    for i in range(n):
        yield i * i

for value in squares(5):
    print(value)
```

Output

```text
0
1
4
9
16
```

# List vs Generator

### List

```python
numbers = [x*x for x in range(5)]
```

Memory:

```text
[0, 1, 4, 9, 16]
```

Everything is stored immediately.

---

### Generator Expression

```python
numbers = (x*x for x in range(5))
```

Nothing is calculated until needed.

---

Example

```python
print(next(numbers))
print(next(numbers))
```

Output

```text
0
1
```

# Infinite Generator

Generators can produce values forever.

```python
def infinite():

    n = 1

    while True:
        yield n
        n += 1

g = infinite()

for _ in range(5):
    print(next(g))
```

Output

```text
1
2
3
4
5
```

A list cannot practically represent an infinite sequence.

# Reading Large Files

Instead of reading an entire file:

```python
with open("data.txt") as file:
    lines = file.readlines()
```

You can process one line at a time:

```python
with open("data.txt") as file:
    for line in file:
        print(line.strip())
```

This behaves similarly to a generator and is much more memory-efficient.

---

# Generator vs Function

Normal Function

```python
def example():
    return 10
```

Runs once and exits.

---

Generator

```python
def example():
    yield 10
```

Pauses and can continue later.

# Generator vs List

| List                           | Generator                                  |
| ------------------------------ | ------------------------------------------ |
| Uses more memory               | Uses less memory                           |
| Stores all values              | Produces values one by one                 |
| Faster for repeated access     | Better for sequential access               |
| Supports indexing              | Does not support indexing                  |
| Can be iterated multiple times | Usually exhausted after one full iteration |

# Generator Expression

Instead of

```python
def squares():

    for i in range(5):
        yield i*i
```

You can write

```python
squares = (i*i for i in range(5))
```

This is similar to a list comprehension:

```python
list_comp = [i*i for i in range(5)]
generator = (i*i for i in range(5))
```

# Real-World Uses

Generators are ideal for:

* Reading large files line by line.
* Processing millions of database records.
* Streaming API responses.
* Producing sensor or IoT data.
* Infinite sequences.
* Data pipelines.

---

# Relationship Between Iterators and Generators

Every **generator is an iterator**, but **not every iterator is a generator**.

A generator automatically implements the iterator protocol (`__iter__()` and `__next__()`), so it works naturally with `for` loops and functions like `next()`.

---

# Key Points to Remember

* A **generator** is a function that uses `yield`.
* `yield` **returns a value and pauses execution**.
* The next call resumes execution from the point where it paused.
* Generators are **memory-efficient** because they produce values on demand.
* A generator can be iterated only once unless you create a new generator object.
* Use generators when working with **large datasets**, **streams**, or **lazy evaluation**.

## Memory Trick

Think of a **list** as a **bucket full of water**:

* You fill the entire bucket before using it.

Think of a **generator** as a **water tap**:

* Water comes out only when you turn the tap.
* You never need to store all the water at once.

```text
List
-----
Creates everything
↓
Stores everything
↓
Uses more memory

Generator
---------
Creates one value
↓
Uses it
↓
Creates next value
↓
Uses very little memory
```

### Closures vs Generators vs Decorators

Since you've now covered all three topics, here's a quick comparison:

| Feature     | Closure                                       | Generator                                         | Decorator                                    |
| ----------- | --------------------------------------------- | ------------------------------------------------- | -------------------------------------------- |
| Purpose     | Remember variables from an enclosing scope    | Produce values lazily, one at a time              | Extend or modify another function's behavior |
| Key Keyword | `nonlocal` (sometimes)                        | `yield`                                           | `@decorator` syntax                          |
| Returns     | A function                                    | A generator object                                | A wrapped function                           |
| Common Uses | Function factories, maintaining private state | Large data processing, streaming, lazy evaluation | Logging, authentication, caching, timing     |
| Built On    | Nested functions                              | Iterator protocol                                 | **Closures**                                 |
