# Python Decorators

A **decorator** in Python is a function that **adds or modifies the behavior of another function (or class) without changing its original source code**. It is one of Python's most powerful features and is widely used for logging, authentication, validation, caching, timing, and more. ([DataCamp][1])

## Why use decorators?

Suppose you have 20 functions, and you want to log every function call.

### Without a decorator

```python
def add(a, b):
    print("Function add() called")
    return a + b

def subtract(a, b):
    print("Function subtract() called")
    return a - b
```

Notice that the logging code is repeated.

### With a decorator

```python
def log_decorator(func):
    def wrapper(*args, **kwargs):
        print(f"Calling function: {func.__name__}")
        return func(*args, **kwargs)
    return wrapper

@log_decorator
def add(a, b):
    return a + b

@log_decorator
def subtract(a, b):
    return a - b

print(add(5, 3))
print(subtract(10, 4))
```

**Output**

```
Calling function: add
8
Calling function: subtract
6
```

The logging functionality is written once and reused.

# How decorators work

When Python sees:

```python
@log_decorator
def greet():
    print("Hello")
```

It is actually doing this:

```python
def greet():
    print("Hello")

greet = log_decorator(greet)
```

The `@` syntax is simply a shortcut. ([Learn Python][2])

# Understanding the structure

```python
def decorator(func):

    def wrapper():
        print("Before function")
        func()
        print("After function")

    return wrapper
```

Here:

* `decorator` receives the original function.
* `wrapper` adds extra behavior.
* `wrapper` calls the original function.
* The decorator returns the wrapper.

Visualization:

```
Original Function
        │
        ▼
   Decorator
        │
        ▼
 Wrapper Function
        │
        ▼
Extra Code
Original Function
Extra Code
```

# Example 1: Before and After execution

```python
def announce(func):

    def wrapper():
        print("Starting...")
        func()
        print("Finished.")

    return wrapper


@announce
def study():
    print("Studying Python")

study()
```

Output

```
Starting...
Studying Python
Finished.
```

# Example 2: Decorator with arguments

The wrapper must accept the same arguments as the original function.

```python
def logger(func):

    def wrapper(*args, **kwargs):
        print("Arguments:", args)
        return func(*args, **kwargs)

    return wrapper


@logger
def multiply(a, b):
    return a * b

print(multiply(5, 4))
```

Output

```
Arguments: (5, 4)
20
```

`*args` and `**kwargs` allow the decorator to work with any function signature.

# Example 3: Timing execution

```python
import time

def timer(func):

    def wrapper(*args, **kwargs):
        start = time.time()
        result = func(*args, **kwargs)
        end = time.time()

        print(f"Execution Time: {end-start:.4f} seconds")
        return result

    return wrapper


@timer
def calculate():
    time.sleep(2)

calculate()
```

Output

```
Execution Time: 2.0002 seconds
```

# Example 4: Authentication

```python
logged_in = True

def login_required(func):

    def wrapper():
        if logged_in:
            return func()
        else:
            print("Access Denied")

    return wrapper


@login_required
def dashboard():
    print("Welcome to Dashboard")

dashboard()
```

Output

```
Welcome to Dashboard
```

# Decorators with parameters

Sometimes the decorator itself needs arguments.

```python
def repeat(times):

    def decorator(func):

        def wrapper(*args, **kwargs):
            for _ in range(times):
                func(*args, **kwargs)

        return wrapper

    return decorator


@repeat(3)
def hello():
    print("Hello")

hello()
```

Output

```
Hello
Hello
Hello
```

Here:

* `repeat(3)` returns a decorator.
* That decorator wraps `hello()`.

# Multiple decorators

```python
def bold(func):

    def wrapper():
        return "<b>" + func() + "</b>"

    return wrapper


def italic(func):

    def wrapper():
        return "<i>" + func() + "</i>"

    return wrapper


@bold
@italic
def text():
    return "Python"

print(text())
```

Output

```
<b><i>Python</i></b>
```

Python applies them from the bottom up:

```
text = bold(italic(text))
```

# Preserving function metadata

Without special handling, a decorator replaces the original function's metadata:

```python
print(add.__name__)
```

Output

```
wrapper
```

To preserve the original name and docstring, use `functools.wraps`:

```python
from functools import wraps

def logger(func):

    @wraps(func)
    def wrapper(*args, **kwargs):
        print("Running")
        return func(*args, **kwargs)

    return wrapper
```

This keeps the original function's metadata intact. ([W3docs][3])

# Common built-in decorators

Python provides several built-in decorators:

| Decorator              | Purpose                                                                              |
| ---------------------- | ------------------------------------------------------------------------------------ |
| `@staticmethod`        | Defines a static method that doesn't receive `self`.                                 |
| `@classmethod`         | Defines a class method that receives `cls`.                                          |
| `@property`            | Allows a method to be accessed like an attribute.                                    |
| `@functools.lru_cache` | Caches function results for faster repeated calls.                                   |
| `@dataclass`           | Automatically generates methods like `__init__()` and `__repr__()` for data classes. |

Example:

```python
class Circle:

    def __init__(self, radius):
        self._radius = radius

    @property
    def area(self):
        return 3.14 * self._radius ** 2

c = Circle(5)
print(c.area)
```

Notice `area` is accessed like an attribute (`c.area`), not a method (`c.area()`).

# Real-world uses

Decorators are commonly used for:

* Logging function calls
* Authentication and authorization
* Input validation
* Measuring execution time
* Caching expensive computations
* Database transaction management
* Retry logic for API calls
* Rate limiting
* Performance monitoring

Frameworks such as Flask, FastAPI, and Django use decorators extensively. For example:

```python
@app.get("/users")
def get_users():
    return ["Alice", "Bob"]
```

Here, `@app.get("/users")` registers the function as an HTTP GET endpoint.

---

# Key points to remember

* A decorator is **a function that takes another function as input and returns a new function**.
* The `@decorator` syntax is shorthand for `function = decorator(function)`.
* Decorators let you **extend behavior without modifying the original code**.
* Use `*args` and `**kwargs` to make decorators work with any function signature.
* Use `functools.wraps` to preserve the original function's metadata.
* Decorators are widely used for cross-cutting concerns such as logging, caching, validation, timing, and authentication.
