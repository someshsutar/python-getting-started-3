# Iterators in Python

An **iterator** is an object that allows you to traverse (iterate over) a collection **one element at a time**.

An iterator remembers its current position, so each call to `next()` returns the next item until there are no more items.

In Python, iterators implement the **iterator protocol**, which consists of two methods:

* `__iter__()` → Returns the iterator object itself.
* `__next__()` → Returns the next value or raises `StopIteration` when there are no more values.

---

# Why do we need iterators?

Suppose you have a list:

```python
numbers = [10, 20, 30, 40]
```

Normally, you iterate like this:

```python
for num in numbers:
    print(num)
```

But internally, Python is actually using an **iterator**.

---

# Creating an Iterator

Use the built-in `iter()` function:

```python
numbers = [10, 20, 30]

iterator = iter(numbers)

print(next(iterator))
print(next(iterator))
print(next(iterator))
```

**Output**

```text
10
20
30
```

Another call:

```python
print(next(iterator))
```

Output

```text
StopIteration
```

The iterator has reached the end of the collection.

---

# How `for` Loop Works Internally

When you write:

```python
numbers = [1, 2, 3]

for num in numbers:
    print(num)
```

Python roughly does this:

```python
iterator = iter(numbers)

while True:
    try:
        num = next(iterator)
        print(num)
    except StopIteration:
        break
```

A `for` loop repeatedly calls `next()` until `StopIteration` is raised.

---

# Iterator Protocol

Every iterator follows these rules:

### `__iter__()`

Returns the iterator itself.

### `__next__()`

Returns the next value.

If no values remain:

```python
raise StopIteration
```

---

# Building Your Own Iterator

Suppose you want an iterator that counts from 1 to 5.

```python
class Counter:

    def __init__(self, limit):
        self.limit = limit
        self.current = 1

    def __iter__(self):
        return self

    def __next__(self):
        if self.current <= self.limit:
            value = self.current
            self.current += 1
            return value
        else:
            raise StopIteration

counter = Counter(5)

for number in counter:
    print(number)
```

**Output**

```text
1
2
3
4
5
```

---

```python
counter = Counter(3)
```

# Iterator State

Unlike a list, an iterator remembers where it is.

```python
numbers = [100, 200, 300]

it = iter(numbers)

print(next(it))
print(next(it))
```

Output

```text
100
200
```

Calling again continues from the current position:

```python
print(next(it))
```

Output

```text
300
```

It does **not** restart from the beginning.

# Iterable vs Iterator

This is one of the most common interview questions.

### Iterable

An object that can produce an iterator.

Examples:

* List
* Tuple
* Dictionary
* String
* Set

Example:

```python
numbers = [1, 2, 3]
```

A list is **iterable** but **not an iterator**.

---

### Iterator

An object that actually returns values one by one.

Example:

```python
it = iter(numbers)
```

Now `it` is an iterator.

---

# Checking Types

```python
numbers = [1, 2, 3]

print(hasattr(numbers, "__iter__"))
print(hasattr(numbers, "__next__"))
```

Output

```text
True
False
```

Lists are iterable but not iterators.

Now:

```python
it = iter(numbers)

print(hasattr(it, "__iter__"))
print(hasattr(it, "__next__"))
```

Output

```text
True
True
```

# Generator vs Iterator

A generator is simply an easier way to create an iterator.

Manual iterator:

```python
class Squares:

    def __init__(self, n):
        self.n = n
        self.i = 0

    def __iter__(self):
        return self

    def __next__(self):
        if self.i < self.n:
            value = self.i * self.i
            self.i += 1
            return value
        raise StopIteration
```

Generator version:

```python
def squares(n):
    for i in range(n):
        yield i * i
```

Both produce the same sequence, but the generator is much shorter because Python automatically implements `__iter__()` and `__next__()`.

---

# Iterator vs Generator

| Iterator                                                         | Generator                             |
| ---------------------------------------------------------------- | ------------------------------------- |
| Created using a class implementing `__iter__()` and `__next__()` | Created using a function with `yield` |
| More code to write                                               | Much simpler                          |
| Full control over iteration                                      | Python manages the state              |
| Can implement complex behavior                                   | Best for sequential value generation  |

# Real-World Examples

Iterators are used in:

* Reading files line by line
* Database cursors
* Streaming APIs
* Processing large datasets
* Lazy evaluation
* Custom collections

Example:

```python
with open("data.txt") as file:
    for line in file:
        print(line)
```

The file object is an iterator that returns one line at a time.

---

# Iterator Lifecycle

```text
Iterable (List, Tuple, String)
             │
          iter()
             │
             ▼
         Iterator
             │
        next()
             │
     Returns next value
             │
        next()
             │
     Returns next value
             │
      No values left
             │
             ▼
      StopIteration
```

---

# Key Points to Remember

* An **iterable** is an object you can loop over.
* An **iterator** is an object that produces values one at a time.
* `iter(obj)` creates an iterator from an iterable.
* `next(iterator)` gets the next item.
* When there are no more items, `StopIteration` is raised.
* Every generator is an iterator, but not every iterator is a generator.
* A `for` loop automatically uses `iter()` and `next()` behind the scenes.

---

# Memory Trick

Imagine a **playlist**:

* The **playlist** itself (all songs) is the **iterable**.
* The **music player** is the **iterator**.
* Pressing **Next** is like calling `next()`.
* When the playlist ends, it's equivalent to `StopIteration`.

## Summary of Related Concepts

| Concept       | Purpose                                     | Keyword/Method             |
| ------------- | ------------------------------------------- | -------------------------- |
| **Iterable**  | Can be looped over                          | `__iter__()`               |
| **Iterator**  | Produces one item at a time                 | `__iter__()`, `__next__()` |
| **Generator** | Simplifies writing iterators                | `yield`                    |
| **Closure**   | Remembers variables from an enclosing scope | Nested functions           |
| **Decorator** | Wraps another function to add behavior      | `@decorator`               |

### Learning Path

```text
Functions
    ↓

    ↓

    ↓

    ↓

    ↓
```

Once you're comfortable with iterators, you'll have a solid understanding of how Python powers `for` loops, comprehensions, generators, and many built-in libraries under the hood.
