# Closures in Python

A **closure** is a function that **remembers the variables from its enclosing (outer) function even after the outer function has finished executing**.

Closures are the foundation of Python decorators.

---

# Why do we need closures?

Normally, local variables of a function disappear once the function returns.

Example:

```python
def outer():
    x = 10
    print(x)

outer()
```

After `outer()` finishes, `x` is destroyed.

But sometimes we want another function to **remember** those variables.

That's where closures come in.

---

# Basic Example

```python
def outer():

    message = "Hello Python"

    def inner():
        print(message)

    return inner

greet = outer()
greet()
```

**Output**

```
Hello Python
```

### What happened?

Step 1

```python
greet = outer()
```

* `outer()` executes.
* `message = "Hello Python"` is created.
* `inner()` is created.
* `outer()` returns `inner`.

Normally, `message` should disappear.

But it doesn't.

Python stores it because `inner()` still needs it.

Step 2

```python
greet()
```

Although `outer()` has already finished, `inner()` still knows the value of `message`.

This "remembering" behavior is called a **closure**.

---

# Visual Representation

```
outer()
│
├── message = "Hello Python"
│
├── create inner()
│
└── return inner
        │
        ▼
message still exists
        │
        ▼
greet()
        │
        ▼
prints "Hello Python"
```

# Another Example

```python
def make_multiplier(n):

    def multiply(x):
        return x * n

    return multiply

double = make_multiplier(2)
triple = make_multiplier(3)

print(double(10))
print(triple(10))
```

Output

```
20
30
```

Notice:

```
double remembers n = 2
triple remembers n = 3
```

Even though `make_multiplier()` has already returned.

---

# Inspecting a Closure

Python stores remembered variables in the `__closure__` attribute.

```python
def outer():

    x = 100

    def inner():
        print(x)

    return inner

func = outer()

print(func.__closure__)
```

Output (similar to):

```
(<cell at 0x7f...: int object at ...>,)
```

You can inspect the stored value:

```python
print(func.__closure__[0].cell_contents)
```

Output

```
100
```

# Closure with Multiple Variables

```python
def person(name, age):

    def display():
        print(name)
        print(age)

    return display

p = person("Alice", 30)
p()
```

Output

```
Alice
30
```

The closure remembers both `name` and `age`.

# Using Closures to Maintain State

Closures can maintain private state without using classes.

```python
def counter():

    count = 0

    def increment():
        nonlocal count
        count += 1
        return count

    return increment

c = counter()

print(c())
print(c())
print(c())
```

Output

```
1
2
3
```

### Why `nonlocal`?

Without `nonlocal`, Python treats `count` inside `increment()` as a new local variable, causing an error because you're trying to assign to it before it has a value.

Using `nonlocal` tells Python to use the `count` from the enclosing function.

---

# Without Closure

```python
count = 0

def increment():
    global count
    count += 1
```

Problems:

* Uses global variables.
* Hard to manage in larger programs.
* Any part of the program can modify `count`.

Closures avoid these issues by keeping the state private.

---

# Closures vs Classes

### Using a Closure

```python
def counter():

    count = 0

    def increment():
        nonlocal count
        count += 1
        return count

    return increment

c = counter()
print(c())
```

### Using a Class

```python
class Counter:

    def __init__(self):
        self.count = 0

    def increment(self):
        self.count += 1
        return self.count

c = Counter()
print(c.increment())
```

Both achieve the same result. Closures are often preferred for simple stateful functions, while classes are better when you need multiple methods or more complex behavior.

---

# Relationship Between Closures and Decorators

Decorators are built using closures.

```python
def logger(func):

    def wrapper(*args, **kwargs):
        print("Calling", func.__name__)
        return func(*args, **kwargs)

    return wrapper
```

Here:

* `logger()` returns `wrapper`.
* `wrapper()` remembers `func`.
* Even after `logger()` has finished, `wrapper()` still has access to `func`.

That remembered reference is the closure.

---

# When to Use Closures

Closures are useful for:

* Building decorators.
* Creating function factories (e.g., `make_multiplier()`).
* Maintaining private state (e.g., counters).
* Configuring callbacks or event handlers.
* Avoiding global variables.

---

# Closure vs Nested Function

These terms are related but not identical.

| Nested Function                             | Closure                                                                                                    |
| ------------------------------------------- | ---------------------------------------------------------------------------------------------------------- |
| A function defined inside another function. | A nested function that remembers variables from its enclosing scope after the outer function has returned. |
| Doesn't necessarily capture variables.      | Captures and retains variables from the enclosing scope.                                                   |

Example of a nested function that is **not** a useful closure:

```python
def outer():
    def inner():
        print("Hello")
    inner()
```

`inner()` is nested, but it isn't returned or used after `outer()` exits, so it doesn't demonstrate the defining behavior of a closure.

---

# Key Points to Remember

* A **closure** is a nested function that remembers variables from its enclosing function.
* The remembered variables remain available even after the enclosing function has finished executing.
* Closures are commonly used to maintain private state without global variables.
* **Decorators are implemented using closures.**
* Use `nonlocal` when a closure needs to modify a variable from its enclosing scope.
* Function factories (like `make_multiplier()`) are a classic use case for closures.

## Simple Memory Trick

Think of a closure like **a backpack**:

* The outer function creates the backpack (the enclosing scope).
* The inner function puts needed variables into it.
* Even after leaving the outer function, the inner function carries that backpack everywhere, so it can still access those variables later.

```
Outer Function
     │
     ▼
Creates variables
     │
     ▼
Inner Function
     │
     ▼
Remembers variables (Closure)
     │
     ▼
Can use them anytime later
```

This ability to "carry" variables from the enclosing scope is what makes closures so powerful and why they're essential for understanding decorators.
