# Threading in Python (Detailed Guide)

## What is Threading?

**Threading** is a technique that allows a program to execute multiple tasks concurrently within the same process. A thread is the smallest unit of execution managed by the operating system.

Think of a process as a company and threads as employees working within the same company. They share the same office (memory) but perform different tasks simultaneously.

### Real-World Example

Imagine you are downloading a file while listening to music and browsing the internet.

* Browser UI remains responsive.
* File download runs in the background.
* Music continues playing.

Each of these activities can be handled by different threads.

---

# Process vs Thread

| Process                  | Thread                  |
| ------------------------ | ----------------------- |
| Independent program      | Smallest execution unit |
| Has its own memory       | Shares process memory   |
| More expensive to create | Lightweight             |
| Communication is slower  | Communication is faster |
| More secure              | Less isolated           |

### Diagram

```text
Process
│
├── Thread 1 (Main Thread)
├── Thread 2 (Download)
├── Thread 3 (Logging)
└── Thread 4 (Notification)

All threads share:

- Variables
- Heap Memory
- Open Files

Each thread has its own:

- Stack
- Program Counter
- Registers
```

# Why Use Threads?

Threading is useful when tasks spend time **waiting** (I/O-bound operations).

Examples:

* Downloading files
* Uploading images
* Reading large files
* Database operations
* Calling REST APIs
* Chat applications
* Email sending
* Background logging

# Creating a Thread

## Method 1: Using target function

```python
import threading
import time

def print_numbers():
    for i in range(5):
        print(i)
        time.sleep(1)

thread = threading.Thread(target=print_numbers)

thread.start()

thread.join()

print("Program Finished")
```

### Output

```text
0
1
2
3
4
Program Finished
```

---

# Understanding `start()`

```python
thread.start()
```

This creates a new thread and invokes the target function.

Never call:

```python
thread.run()
```

because it executes in the current thread instead of creating a new one.

---

# Understanding `join()`

`join()` tells the main thread to wait until another thread completes.

Without `join()`:

```python
thread.start()

print("Finished")
```

Possible Output

```text
Finished
0
1
2
3
4
```

With `join()`:

```python
thread.start()
thread.join()

print("Finished")
```

Output

```text
0
1
2
3
4
Finished
```

# Running Multiple Threads

```python
import threading
import time

def task(name):
    for i in range(3):
        print(name, i)
        time.sleep(1)

t1 = threading.Thread(target=task, args=("Thread-1",))
t2 = threading.Thread(target=task, args=("Thread-2",))

t1.start()
t2.start()

t1.join()
t2.join()

print("Completed")
```

Output (order may vary):

```text
Thread-1 0
Thread-2 0
Thread-1 1
Thread-2 1
Thread-1 2
Thread-2 2
Completed
```

# Daemon Threads

A daemon thread runs in the background and automatically exits when all non-daemon threads finish.

Examples:

* Logging
* Monitoring
* Auto-save
* Background cleanup

```python
import threading
import time

def monitor():
    while True:
        print("Monitoring...")
        time.sleep(2)

thread = threading.Thread(target=monitor, daemon=True)

thread.start()

time.sleep(5)

print("Main Thread Finished")
```

The daemon thread stops automatically when the main thread exits.

# Thread Communication

Threads can communicate using shared variables.

Example:

```python
import threading

counter = 0

def increment():
    global counter
    counter += 1

threads = []

for _ in range(100):
    t = threading.Thread(target=increment)
    threads.append(t)
    t.start()

for t in threads:
    t.join()

print(counter)
```

This may not always print `100` because of race conditions.

---

# Race Condition

A race condition occurs when multiple threads access and modify shared data at the same time, leading to unpredictable results.

Example:

```python
counter += 1
```

This seemingly simple statement actually performs three steps:

1. Read `counter`
2. Add `1`
3. Write back

If two threads execute these steps simultaneously, one update may overwrite the other.

---

# Lock (Mutual Exclusion)

A `Lock` ensures that only one thread accesses a critical section at a time.

```python
import threading

counter = 0

lock = threading.Lock()

def increment():
    global counter

    for _ in range(100000):
        with lock:
            counter += 1

threads = []

for _ in range(5):
    t = threading.Thread(target=increment)
    threads.append(t)
    t.start()

for t in threads:
    t.join()

print(counter)
```

Output

```text
500000
```

---

# Without Lock vs With Lock

Without Lock

```text
Thread A reads 10
Thread B reads 10

Thread A writes 11
Thread B writes 11

Expected = 12

Actual = 11
```

With Lock

```text
Thread A acquires lock

Reads 10

Writes 11

Releases lock

Thread B acquires lock

Reads 11

Writes 12

Releases lock
```

# RLock (Reentrant Lock)

Allows the same thread to acquire the same lock multiple times.

```python
import threading

lock = threading.RLock()

def inner():
    with lock:
        print("Inner")

def outer():
    with lock:
        inner()

outer()
```

# Semaphore

Controls how many threads can access a resource simultaneously.

Example: Only three customers can use an ATM room at the same time.

```python
import threading
import time

sem = threading.Semaphore(3)

def worker(i):
    with sem:
        print(f"Worker {i} started")
        time.sleep(2)
        print(f"Worker {i} finished")

for i in range(10):
    threading.Thread(target=worker, args=(i,)).start()
```

# Event

One thread signals another thread when an event occurs.

```python
import threading
import time

event = threading.Event()

def waiter():
    print("Waiting...")
    event.wait()
    print("Started")

threading.Thread(target=waiter).start()

time.sleep(3)

event.set()
```

# Condition

Allows threads to wait until notified.

Example:

Producer-Consumer

```python
import threading

condition = threading.Condition()
items = []

def producer():
    with condition:
        items.append("Apple")
        condition.notify()

def consumer():
    with condition:
        while not items:
            condition.wait()
        print(items.pop())

threading.Thread(target=consumer).start()
threading.Thread(target=producer).start()
```

# ThreadPoolExecutor

A thread pool reuses a fixed number of threads instead of creating new ones for every task.

```python
from concurrent.futures import ThreadPoolExecutor
import time

def square(n):
    time.sleep(1)
    return n * n

numbers = [1, 2, 3, 4, 5]

with ThreadPoolExecutor(max_workers=3) as executor:
    results = executor.map(square, numbers)

for result in results:
    print(result)
```

Output

```text
1
4
9
16
25
```

# Real-World Example: Downloading Multiple Files

```python
import threading
import time

files = [
    "Python.pdf",
    "AI.docx",
    "Image.png",
    "Video.mp4"
]

def download(file):
    print(f"Downloading {file}")
    time.sleep(3)
    print(f"Completed {file}")

threads = []

for file in files:
    t = threading.Thread(target=download, args=(file,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

print("All Downloads Finished")
```

Without threading, the downloads would occur one after another. With threading, they overlap, reducing the total time when the work is I/O-bound.

# Advantages of Threading

* Improves responsiveness for applications.
* Efficient for I/O-bound tasks.
* Lightweight compared to processes.
* Threads share memory, making communication easy.
* Suitable for background services and network applications.

---

# Limitations

* **Global Interpreter Lock (GIL):** In CPython, only one thread executes Python bytecode at a time, so threading does not speed up CPU-bound tasks.
* Shared memory can lead to race conditions if not synchronized properly.
* Debugging concurrent code can be challenging.
* Deadlocks can occur if locks are not managed carefully.

---

# When to Use Threading

| Scenario                  | Use Threading? | Reason                                 |
| ------------------------- | -------------- | -------------------------------------- |
| Downloading files         | Yes          | I/O-bound                              |
| REST API calls            | Yes          | Waiting for network responses          |
| Reading multiple files    | Yes          | Disk I/O                               |
| Database operations       | Yes          | Waiting for queries                    |
| GUI applications          | Yes          | Keep the UI responsive                 |
| Image processing          | No           | CPU-bound; use multiprocessing instead |
| Video encoding            | No           | CPU-bound                              |
| Machine learning training | No           | CPU-bound                              |

# Summary

| Concept              | Description                                         |
| -------------------- | --------------------------------------------------- |
| Thread               | Smallest unit of execution within a process         |
| `Thread`             | Creates a new thread                                |
| `start()`            | Starts execution in a new thread                    |
| `join()`             | Waits for a thread to finish                        |
| `daemon`             | Background thread that ends with the main program   |
| `Lock`               | Prevents race conditions                            |
| `RLock`              | Reentrant lock for the same thread                  |
| `Semaphore`          | Limits concurrent access to a resource              |
| `Event`              | Signals between threads                             |
| `Condition`          | Coordinates threads waiting for specific conditions |
| `ThreadPoolExecutor` | Efficiently manages a pool of reusable threads      |

### Best Practices

* Use threads primarily for **I/O-bound** work.
* Always synchronize access to shared mutable data with `Lock` or higher-level synchronization primitives.
* Prefer `ThreadPoolExecutor` over manually creating many threads for scalable applications.
* Use `join()` when the main thread depends on worker threads completing.
* For CPU-intensive work, prefer the `multiprocessing` module to bypass the GIL.
