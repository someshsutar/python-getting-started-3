# Async/Await for I/O-Bound Tasks in Python

## What is Asynchronous Programming?

**Asynchronous programming** is a programming model that allows a program to perform other work while waiting for slow operations (such as network requests, database queries, or file operations) to complete.

Instead of blocking the program during waiting periods, Python's **`asyncio`** framework suspends the current task and allows another task to run.

> **Key Idea:** Async programming is **single-threaded concurrency**. It is designed for **I/O-bound** tasks, not CPU-intensive computations.

---

# What are I/O-Bound Tasks?

I/O-bound tasks spend most of their time **waiting** for external resources rather than performing calculations.

Examples include:

* Downloading web pages
* Calling REST APIs
* Reading or writing files
* Database queries
* Sending emails
* Chat applications
* Web servers
* Cloud storage operations

---

# Why Not Use Normal Functions?

Consider downloading three files.

### Synchronous Approach

```text
Download File 1 (2 sec)
↓
Download File 2 (2 sec)
↓
Download File 3 (2 sec)

Total Time = 6 seconds
```

The program waits for each download to finish before starting the next one.

---

### Asynchronous Approach

```text
Start File 1
↓
Waiting...

Start File 2
↓
Waiting...

Start File 3
↓
Waiting...

All complete together

Total Time ≈ 2 seconds
```

The program starts all downloads and switches between them while each is waiting.

---

# Asyncio Architecture

```text
                    Event Loop
                         │
      ┌──────────────────┼──────────────────┐
      │                  │                  │
 Task 1             Task 2             Task 3
(API Call)      (DB Query)      (File Download)
      │                  │                  │
 Waiting           Waiting           Waiting
      │                  │                  │
 Event Loop runs another ready task
```

The **Event Loop** continuously checks which task is ready to run.

# Important Terminology

| Term       | Meaning                                           |
| ---------- | ------------------------------------------------- |
| Coroutine  | Function declared using `async def`               |
| Event Loop | Scheduler that runs asynchronous tasks            |
| await      | Suspends a coroutine until an operation completes |
| Task       | A scheduled coroutine                             |
| Future     | Placeholder for a result available later          |

# Creating a Coroutine

```python
import asyncio

async def hello():
    print("Hello")
```

Notice the use of:

```python
async def
```

This creates a **coroutine**, not a normal function.

Calling it directly:

```python
hello()
```

returns:

```text
<coroutine object hello at ...>
```

It does **not** execute the function.

---

# Running a Coroutine

```python
import asyncio

async def hello():
    print("Hello Async World")

asyncio.run(hello())
```

Output

```text
Hello Async World
```

`asyncio.run()`:

* Creates the event loop.
* Executes the coroutine.
* Closes the event loop.

# Understanding `await`

Suppose downloading takes 2 seconds.

```python
import asyncio

async def download():
    print("Downloading...")
    await asyncio.sleep(2)
    print("Completed")

asyncio.run(download())
```

Output

```text
Downloading...
(wait 2 seconds)
Completed
```

`await` means:

> "Pause this coroutine and allow another coroutine to execute."

---

# `time.sleep()` vs `asyncio.sleep()`

### Blocking

```python
import time

time.sleep(5)
```

Everything stops for 5 seconds.

---

### Non-blocking

```python
await asyncio.sleep(5)
```

Only the current coroutine pauses. The event loop continues running other coroutines.

# Running Multiple Coroutines

```python
import asyncio

async def task(name):
    print(f"{name} Started")

    await asyncio.sleep(2)

    print(f"{name} Finished")

async def main():

    await asyncio.gather(
        task("Task-1"),
        task("Task-2"),
        task("Task-3")
    )

asyncio.run(main())
```

Output

```text
Task-1 Started
Task-2 Started
Task-3 Started

(wait 2 seconds)

Task-1 Finished
Task-2 Finished
Task-3 Finished
```

Total execution time ≈ 2 seconds.

# Sequential vs Concurrent

### Sequential

```python
async def main():
    await task("A")
    await task("B")
    await task("C")
```

Timeline

```text
A → 2 sec

B → 2 sec

C → 2 sec

Total = 6 sec
```

---

### Concurrent

```python
await asyncio.gather(
    task("A"),
    task("B"),
    task("C")
)
```

Timeline

```text
A
B
C

↓

All waiting together

↓

Finish together

Total ≈ 2 sec
```

# Creating Tasks

```python
import asyncio

async def work(number):
    await asyncio.sleep(2)
    print(number)

async def main():

    task1 = asyncio.create_task(work(1))
    task2 = asyncio.create_task(work(2))

    await task1
    await task2

asyncio.run(main())
```

`create_task()` schedules the coroutine immediately.

---

# Waiting for Multiple Tasks

```python
await asyncio.gather(
    task1,
    task2,
    task3
)
```

`gather()` waits until every task finishes.

# Real-World Example: Downloading Files

```python
import asyncio

files = [
    "Python.pdf",
    "AI.pdf",
    "MachineLearning.pdf"
]

async def download(file):

    print(f"Downloading {file}")

    await asyncio.sleep(3)

    print(f"Completed {file}")

async def main():

    tasks = [
        download(file)
        for file in files
    ]

    await asyncio.gather(*tasks)

asyncio.run(main())
```

Without async:

* 3 + 3 + 3 = 9 seconds

With async:

* Approximately 3 seconds

# Real-World Example: Calling Multiple APIs

```python
import asyncio
import random

async def fetch_api(api_name):
    print(f"Calling {api_name}...")
    await asyncio.sleep(random.randint(1, 3))
    return f"{api_name} response"

async def main():
    responses = await asyncio.gather(
        fetch_api("Weather API"),
        fetch_api("Maps API"),
        fetch_api("Payment API")
    )

    for response in responses:
        print(response)

asyncio.run(main())
```

Sample Output

```text
Calling Weather API...
Calling Maps API...
Calling Payment API...

Weather API response
Maps API response
Payment API response
```

# Async Producer-Consumer

```python
import asyncio

queue = asyncio.Queue()

async def producer():
    for i in range(5):
        await queue.put(i)
        print(f"Produced {i}")

async def consumer():
    while True:
        item = await queue.get()
        print(f"Consumed {item}")
        queue.task_done()

async def main():
    consumer_task = asyncio.create_task(consumer())

    await producer()

    await queue.join()

    consumer_task.cancel()

asyncio.run(main())
```

# Running Blocking Code

Suppose you already have a blocking function.

```python
import time

def slow_function():
    time.sleep(5)
```

Run it asynchronously:

```python
import asyncio

async def main():

    await asyncio.to_thread(slow_function)

asyncio.run(main())
```

`asyncio.to_thread()` runs the blocking function in a separate thread without blocking the event loop.

# Exception Handling

```python
import asyncio

async def divide(a, b):
    return a / b

async def main():
    try:
        result = await divide(10, 0)
        print(result)
    except ZeroDivisionError:
        print("Cannot divide by zero")

asyncio.run(main())
```

# Timeout Example

```python
import asyncio

async def work():
    await asyncio.sleep(5)

async def main():
    try:
        await asyncio.wait_for(work(), timeout=2)
    except asyncio.TimeoutError:
        print("Timeout!")

asyncio.run(main())
```

# Cancellation

```python
import asyncio

async def worker():
    try:
        while True:
            print("Working...")
            await asyncio.sleep(1)
    except asyncio.CancelledError:
        print("Cancelled!")

async def main():
    task = asyncio.create_task(worker())

    await asyncio.sleep(3)

    task.cancel()

    await task

asyncio.run(main())
```

# Advantages

* Excellent for thousands of concurrent I/O operations.
* Low memory usage compared to threads.
* Fast context switching.
* No thread synchronization issues for purely async code.
* Scales well for web servers and network applications.

---

# Limitations

* Not suitable for CPU-bound work.
* Requires async-compatible libraries (e.g., `aiohttp`, `asyncpg`).
* Blocking functions (such as `time.sleep()`) freeze the event loop unless offloaded.
* Async code can be more complex to understand and debug.

---

# Asyncio vs Threading vs Multiprocessing

| Feature            | Asyncio                                 | Threading | Multiprocessing                  |
| ------------------ | --------------------------------------- | --------- | -------------------------------- |
| Best For           | I/O-bound                               | I/O-bound | CPU-bound                        |
| Uses Threads       | No (single-threaded event loop)         | Yes       | No                               |
| Uses Processes     | No                                      | No        | Yes                              |
| GIL Issue          | Not applicable for async scheduling     | Yes       | No                               |
| Multiple CPU Cores | No                                      | No        | Yes                              |
| Memory Usage       | Very low                                | Moderate  | High                             |
| Scalability        | Excellent for many concurrent I/O tasks | Good      | Excellent for heavy computations |

| Task                                  | Use Asyncio? |
| ------------------------------------- | ------------ |
| REST API calls                        | Yes        |
| Web scraping                          | Yes        |
| Chat applications                     | Yes        |
| WebSocket servers                     | Yes        |
| Database queries (with async drivers) | Yes        |
| File downloads/uploads                | Yes        |
| Image processing                      | No         |
| Video encoding                        | No         |
| Machine learning                      | No         |
| Mathematical calculations             | No         |

---

# Best Practices

* Use `asyncio.run()` as the main entry point.
* Prefer `asyncio.gather()` to run independent I/O tasks concurrently.
* Never use `time.sleep()` inside a coroutine; use `await asyncio.sleep()`.
* Use `asyncio.create_task()` for background asynchronous work.
* Use `asyncio.to_thread()` for legacy blocking functions.
* Keep CPU-intensive work out of the event loop; use `multiprocessing` or a process pool instead.

---

# Summary

| Concept                 | Description                                                 |
| ----------------------- | ----------------------------------------------------------- |
| `async def`             | Defines a coroutine                                         |
| `await`                 | Suspends the current coroutine until an awaitable completes |
| `asyncio.run()`         | Starts and manages the event loop                           |
| `asyncio.sleep()`       | Non-blocking delay                                          |
| `asyncio.gather()`      | Runs multiple coroutines concurrently and waits for all     |
| `asyncio.create_task()` | Schedules a coroutine to run concurrently                   |
| `asyncio.wait_for()`    | Adds a timeout to an awaitable                              |
| `asyncio.to_thread()`   | Runs blocking code in a separate thread                     |
| `asyncio.Queue`         | Asynchronous producer-consumer queue                        |

## Rule of Thumb

* **Many network requests, REST APIs, WebSockets, chat servers, or cloud services?** → Use **`asyncio`**.
* **Blocking file or database operations without async libraries?** → Use **`threading`** or **`asyncio.to_thread()`**.
* **Heavy computations, image processing, or machine learning?** → Use **`multiprocessing`**.

In modern Python applications such as **FastAPI**, **aiohttp**, and **async web services**, `async`/`await` is the preferred approach for building highly scalable, I/O-efficient systems.
