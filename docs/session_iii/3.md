# Multiprocessing for CPU-Bound Tasks in Python

## What is Multiprocessing?

**Multiprocessing** is a technique where multiple **processes** execute simultaneously. Each process has its own Python interpreter and memory space, allowing true parallel execution on multiple CPU cores.

Unlike threads, multiprocessing is **not affected by Python's Global Interpreter Lock (GIL)**, making it the preferred choice for CPU-intensive applications.

---

# Why Use Multiprocessing?

For CPU-bound tasks, threading provides little to no speed improvement because of the GIL. Multiprocessing overcomes this by running each task in a separate process.

### CPU-Bound Examples

* Image processing
* Video encoding
* Data compression
* Machine learning model training
* Scientific simulations
* Cryptography
* Matrix multiplication
* Prime number generation
* Password hashing

---

# Multiprocessing vs Threading

| Feature         | Threading                       | Multiprocessing                  |
| --------------- | ------------------------------- | -------------------------------- |
| Unit            | Threads                         | Processes                        |
| Memory          | Shared                          | Separate                         |
| GIL             | Yes                             | No                               |
| CPU Utilization | Single core for Python bytecode | Multiple CPU cores               |
| Best For        | I/O-bound tasks                 | CPU-bound tasks                  |
| Communication   | Easy (shared memory)            | Requires IPC (Queue, Pipe, etc.) |
| Creation Cost   | Low                             | Higher                           |

# Process Architecture

```text
                    Operating System
                           │
         ┌─────────────────┼─────────────────┐
         │                 │                 │
    Process 1         Process 2         Process 3
    (Core 1)          (Core 2)          (Core 3)
         │                 │                 │
 Python Interpreter  Python Interpreter  Python Interpreter
         │                 │                 │
     Own Memory        Own Memory        Own Memory
```

Each process:

* Has its own memory.
* Has its own GIL.
* Can execute simultaneously on different CPU cores.

# Creating a Process

```python
from multiprocessing import Process
import os

def worker():
    print(f"Worker Process ID: {os.getpid()}")

if __name__ == "__main__":
    p = Process(target=worker)

    p.start()
    p.join()

    print("Main Process Finished")
```

### Output

```text
Worker Process ID: 12453
Main Process Finished
```

---

# Understanding `start()` and `join()`

```python
p.start()
```

Starts the child process.

```python
p.join()
```

Waits for the child process to finish.

Without `join()`, the main process may terminate before child processes complete.

---

# Running Multiple Processes

```python
from multiprocessing import Process
import time
import os

def task(number):
    print(f"Process {os.getpid()} computing {number}")
    time.sleep(2)

if __name__ == "__main__":
    processes = []

    for i in range(5):
        p = Process(target=task, args=(i,))
        processes.append(p)
        p.start()

    for p in processes:
        p.join()

    print("All processes completed.")
```

# Using `Pool`

Instead of creating processes manually, use a process pool.

```python
from multiprocessing import Pool

def cube(n):
    return n ** 3

if __name__ == "__main__":
    numbers = [1,2,3,4,5]

    with Pool() as pool:
        result = pool.map(cube, numbers)

    print(result)
```

Output

```text
[1, 8, 27, 64, 125]
```

---

# Pool with Multiple CPU Cores

```python
from multiprocessing import Pool

def factorial(n):
    result = 1

    for i in range(1, n+1):
        result *= i

    return result

if __name__ == "__main__":
    numbers = [5,6,7,8,9]

    with Pool(processes=4) as pool:
        results = pool.map(factorial, numbers)

    print(results)
```

# Using `apply()`

Runs one task and waits for completion.

```python
from multiprocessing import Pool

def add(numbers):
    return numbers[0] + numbers[1]

if __name__ == "__main__":
    with Pool(2) as pool:
        result = pool.apply(add, args=([10,20],))

    print(result)
```

---

# Using `apply_async()`

Runs tasks asynchronously.

```python
from multiprocessing import Pool

def multiply(n):
    return n * n

if __name__ == "__main__":
    with Pool(4) as pool:
        result = pool.apply_async(multiply, (10,))
        print(result.get())
```

---

# Using `starmap()`

Useful for functions with multiple arguments.

```python
from multiprocessing import Pool

def add(a, b):
    return a + b

if __name__ == "__main__":
    values = [
        (1,2),
        (3,4),
        (5,6)
    ]

    with Pool() as pool:
        result = pool.starmap(add, values)

    print(result)
```

Output

```text
[3, 7, 11]
```

# Sharing Data Between Processes

Processes do not share memory automatically. Use IPC (Inter-Process Communication).

### Queue Example

```python
from multiprocessing import Process, Queue

def worker(queue):
    queue.put("Hello from Process")

if __name__ == "__main__":
    q = Queue()

    p = Process(target=worker, args=(q,))
    p.start()

    print(q.get())

    p.join()
```

---

# Pipe Example

```python
from multiprocessing import Process, Pipe

def worker(conn):
    conn.send("Data Sent")
    conn.close()

if __name__ == "__main__":
    parent, child = Pipe()

    p = Process(target=worker, args=(child,))
    p.start()

    print(parent.recv())

    p.join()
```

# Shared Variables

Use `Value` or `Array` to share simple data.

```python
from multiprocessing import Process, Value

def increment(counter):
    counter.value += 1

if __name__ == "__main__":
    counter = Value('i', 0)

    p = Process(target=increment, args=(counter,))
    p.start()
    p.join()

    print(counter.value)
```

---

# Shared List

```python
from multiprocessing import Manager, Process

def worker(shared):
    shared.append("Python")

if __name__ == "__main__":
    with Manager() as manager:
        data = manager.list()

        p = Process(target=worker, args=(data,))
        p.start()
        p.join()

        print(list(data))
```

# Synchronization with Lock

```python
from multiprocessing import Process, Lock, Value

def increment(counter, lock):
    for _ in range(100000):
        with lock:
            counter.value += 1

if __name__ == "__main__":
    counter = Value('i', 0)
    lock = Lock()

    processes = []

    for _ in range(5):
        p = Process(target=increment, args=(counter, lock))
        processes.append(p)
        p.start()

    for p in processes:
        p.join()

    print(counter.value)
```

# Real-World Example: Prime Number Calculation

```python
from multiprocessing import Pool
import math

def is_prime(n):
    if n < 2:
        return False

    for i in range(2, int(math.sqrt(n)) + 1):
        if n % i == 0:
            return False

    return True

if __name__ == "__main__":
    numbers = range(100000, 100100)

    with Pool() as pool:
        primes = pool.map(is_prime, numbers)

    for n, prime in zip(numbers, primes):
        if prime:
            print(n)
```

Each process checks different numbers in parallel, significantly reducing computation time.

# Why `if __name__ == "__main__":` Is Required

On Windows and macOS (and increasingly recommended everywhere), child processes import the main module to start execution.

Without this guard:

```python
from multiprocessing import Process

def task():
    print("Running")

p = Process(target=task)
p.start()
```

the module can recursively create new processes, leading to errors or infinite process spawning.

Correct usage:

```python
if __name__ == "__main__":
    p = Process(target=task)
    p.start()
    p.join()
```

# Advantages

* True parallel execution.
* Utilizes multiple CPU cores.
* Not limited by the GIL.
* Ideal for CPU-intensive computations.
* Better performance for scientific and mathematical workloads.

---

# Limitations

* Higher memory usage than threading.
* Process creation is slower.
* Data sharing requires IPC mechanisms.
* Frequent inter-process communication can reduce performance.
* Not ideal for lightweight or I/O-bound tasks.

---

# Best Practices

* Use `multiprocessing.Pool` or `concurrent.futures.ProcessPoolExecutor` for processing collections of tasks.
* Always protect the entry point with `if __name__ == "__main__":`.
* Minimize data transferred between processes.
* Prefer immutable inputs and return computed results instead of sharing state.
* Use multiprocessing only when the computation is substantial enough to offset process creation overhead.

# Summary

| Feature           | Description                                           |
| ----------------- | ----------------------------------------------------- |
| `Process`         | Creates a child process                               |
| `start()`         | Starts execution of the child process                 |
| `join()`          | Waits for the child process to finish                 |
| `Pool`            | Manages a pool of worker processes                    |
| `map()`           | Applies a function to each item in an iterable        |
| `apply()`         | Executes a single task synchronously                  |
| `apply_async()`   | Executes a task asynchronously                        |
| `starmap()`       | Maps a function over iterables of argument tuples     |
| `Queue`           | Exchanges messages between processes                  |
| `Pipe`            | Two-way communication between two processes           |
| `Value` / `Array` | Shared primitive data                                 |
| `Manager`         | Shared high-level objects like lists and dictionaries |
| `Lock`            | Synchronizes access to shared resources               |

### Rule of Thumb

| Task Type                      | Recommended Approach     |
| ------------------------------ | ------------------------ |
| Downloading files              | `threading` or `asyncio` |
| Calling REST APIs              | `asyncio` or `threading` |
| Database queries               | `threading` or `asyncio` |
| Image/video processing         | `multiprocessing`        |
| Scientific computing           | `multiprocessing`        |
| Machine learning preprocessing | `multiprocessing`        |
| Large numerical calculations   | `multiprocessing`        |

For CPU-bound work in Python, **multiprocessing** is generally the preferred solution because it enables true parallelism by using multiple CPU cores and avoiding the limitations imposed by the GIL.
