# Concurrency in Python

**Concurrency** is the ability of a program to make progress on multiple tasks during the same period of time. It does **not necessarily mean multiple tasks execute simultaneously** (that's parallelism). Concurrency is mainly about efficiently managing multiple tasks, especially when they spend time waiting for I/O operations.

Python provides several ways to achieve concurrency:

1. Threading
2. Multiprocessing
3. Asyncio (Asynchronous Programming)
4. Concurrent Futures

---

# Concurrency vs Parallelism

| Concurrency                           | Parallelism                             |
| ------------------------------------- | --------------------------------------- |
| Multiple tasks make progress together | Multiple tasks execute at the same time |
| Best for I/O-bound tasks              | Best for CPU-bound tasks                |
| Usually uses threads or async         | Usually uses multiple processes         |
| Doesn't always use multiple CPU cores | Uses multiple CPU cores                 |

### Example

Imagine a chef cooking food.

**Concurrency**

* The chef starts boiling rice.
* While waiting, chops vegetables.
* Then stirs curry.
* One chef handles multiple tasks.

**Parallelism**

* Three chefs cook three dishes simultaneously.

# Python Global Interpreter Lock (GIL)

One important concept in Python is the **Global Interpreter Lock (GIL)**.

The GIL allows **only one thread to execute Python bytecode at a time** within a single Python process.

Because of the GIL:

* Threads are excellent for **I/O-bound** tasks.
* Threads do **not** improve performance for **CPU-bound** tasks.
* Multiprocessing avoids the GIL because each process has its own Python interpreter.

# 1. Threading

Multiple threads share the same process memory.

Suitable for:

* File operations
* Network requests
* API calls
* Database operations

## Example: Downloading Files

```python
import threading
import time

def download(file):
    print(f"Downloading {file}...")
    time.sleep(2)
    print(f"{file} downloaded")

threads = []

files = ["File1", "File2", "File3"]

for file in files:
    t = threading.Thread(target=download, args=(file,))
    threads.append(t)
    t.start()

for t in threads:
    t.join()

print("All downloads completed.")
```

### Output

```
Downloading File1...
Downloading File2...
Downloading File3...
File1 downloaded
File2 downloaded
File3 downloaded
All downloads completed.
```

# Thread Synchronization using Lock

Multiple threads sharing data may cause race conditions.

```python
import threading

counter = 0
lock = threading.Lock()

def increment():
    global counter

    for _ in range(100000):
        with lock:
            counter += 1

threads = []

for _ in range(5):
    t = threading.Thread(target=increment)
    threads.append(t)
    t.start()

for t in threads:
    t.join()

print(counter)
```

Without the lock, the final counter value may be incorrect.

# 2. Multiprocessing

Creates multiple independent Python processes.

Advantages:

* Uses multiple CPU cores
* No GIL limitation
* Best for CPU-intensive tasks

## Example

```python
from multiprocessing import Process
import time

def square(number):
    print(f"Square of {number} = {number * number}")
    time.sleep(2)

processes = []

for i in range(5):
    p = Process(target=square, args=(i,))
    processes.append(p)
    p.start()

for p in processes:
    p.join()

print("Finished")
```

---

# Multiprocessing with Pool

```python
from multiprocessing import Pool

def cube(n):
    return n ** 3

numbers = [1,2,3,4,5]

with Pool() as pool:
    result = pool.map(cube, numbers)

print(result)
```

Output

```
[1, 8, 27, 64, 125]
```

# 3. Asyncio

Asyncio is Python's built-in framework for asynchronous programming.

Ideal for:

* Web APIs
* Chat servers
* Network programming
* High-performance I/O

Uses:

* `async`
* `await`
* Event Loop

---

## Example

```python
import asyncio

async def download(file):
    print(f"Downloading {file}")
    await asyncio.sleep(2)
    print(f"{file} completed")

async def main():
    await asyncio.gather(
        download("File1"),
        download("File2"),
        download("File3")
    )

asyncio.run(main())
```

Output

```
Downloading File1
Downloading File2
Downloading File3
File1 completed
File2 completed
File3 completed
```

---

# Event Loop

The event loop schedules and executes asynchronous tasks.

```
Task1 ---> Waiting
            |
Task2 ---> Running
            |
Task3 ---> Waiting

(Event Loop keeps switching)
```

---

# async vs await

```python
async def fetch_data():
    await asyncio.sleep(2)
    return "Done"
```

* `async` declares a coroutine.
* `await` pauses the coroutine until the awaited operation completes, allowing other coroutines to run.

# 4. concurrent.futures

Provides a high-level interface for running tasks using threads or processes.

### ThreadPoolExecutor Example

```python
from concurrent.futures import ThreadPoolExecutor
import time

def task(name):
    time.sleep(2)
    return f"{name} completed"

with ThreadPoolExecutor(max_workers=3) as executor:
    results = executor.map(task, ["A", "B", "C"])

for result in results:
    print(result)
```

---

### ProcessPoolExecutor Example

```python
from concurrent.futures import ProcessPoolExecutor

def factorial(n):
    if n == 0:
        return 1

    result = 1
    for i in range(1, n + 1):
        result *= i

    return result

numbers = [5,6,7,8]

with ProcessPoolExecutor() as executor:
    results = executor.map(factorial, numbers)

for result in results:
    print(result)
```

# Comparison Table

| Feature            | Threading | Multiprocessing | Asyncio                         |
| ------------------ | --------- | --------------- | ------------------------------- |
| Uses Threads       | Yes        | No               | No                               |
| Uses Processes     | No         | Yes               | No                               |
| Uses Event Loop    | No         | Yes               | Yes                               |
| Best For           | I/O-bound | CPU-bound       | High-concurrency I/O            |
| Affected by GIL    | Yes       | No              | No (single-threaded event loop) |
| Shares Memory      | Yes       | No              | Yes (single process)            |
| Multiple CPU Cores | No        | Yes             | No                              |

# Mini Project: Concurrent File Backup System

This example demonstrates all three approaches with a practical scenario.

```python
import os
import shutil
import asyncio
from concurrent.futures import ThreadPoolExecutor

SOURCE = "source_files"
DESTINATION = "backup"

os.makedirs(DESTINATION, exist_ok=True)

def copy_file(file_name):
    src = os.path.join(SOURCE, file_name)
    dst = os.path.join(DESTINATION, file_name)

    shutil.copy(src, dst)
    print(f"Copied {file_name}")

async def backup():
    files = os.listdir(SOURCE)

    loop = asyncio.get_running_loop()

    with ThreadPoolExecutor(max_workers=4) as executor:
        tasks = [
            loop.run_in_executor(executor, copy_file, file)
            for file in files
        ]

        await asyncio.gather(*tasks)

asyncio.run(backup())
```

**Why this design?**

* `asyncio` orchestrates the overall workflow.
* `ThreadPoolExecutor` handles blocking file copy operations concurrently.
* This approach efficiently backs up many files without blocking the main event loop.

# Summary

| Method               | Best Used For                         | Advantages                                             | Limitations                                                  |
| -------------------- | ------------------------------------- | ------------------------------------------------------ | ------------------------------------------------------------ |
| `threading`          | I/O-bound tasks                       | Simple, shared memory, lightweight                     | Limited by the GIL for CPU-bound work                        |
| `multiprocessing`    | CPU-bound tasks                       | True parallelism across CPU cores                      | Higher memory usage and inter-process communication overhead |
| `asyncio`            | Large numbers of I/O-bound operations | Very efficient, scalable, low overhead                 | Requires async-compatible libraries and programming style    |
| `concurrent.futures` | General-purpose concurrency           | High-level, easy-to-use API over threads and processes | Less control than lower-level concurrency primitives         |

### Rule of Thumb

* **Need to handle many network requests, APIs, or sockets?** → Use **`asyncio`**.
* **Need to perform blocking I/O (files, databases, HTTP libraries without async support)?** → Use **`threading`** or **`ThreadPoolExecutor`**.
* **Need to maximize CPU usage for heavy computations?** → Use **`multiprocessing`** or **`ProcessPoolExecutor`**.
* **Want a simple and modern concurrency API?** → Start with **`concurrent.futures`**, choosing a thread or process pool based on whether the workload is I/O-bound or CPU-bound.
