# List Comprehensions vs Generator Expressions in Python

Both **List Comprehensions** and **Generator Expressions** provide a concise way to create sequences in Python. The key difference is **when values are generated**.

| Feature      | List Comprehension                  | Generator Expression                                              |
| ------------ | ----------------------------------- | ----------------------------------------------------------------- |
| Syntax       | `[expression for item in iterable]` | `(expression for item in iterable)`                               |
| Result       | Returns a **list**                  | Returns a **generator object**                                    |
| Memory Usage | Higher (stores all elements)        | Lower (generates one element at a time)                           |
| Execution    | Eager (immediate)                   | Lazy (on demand)                                                  |
| Best For     | Small to medium datasets            | Large or infinite datasets                                        |
| Reusable     | Yes                                 | No (once exhausted)                                               |
| Speed        | Faster when all values are needed   | Slower per item due to lazy evaluation, but more memory-efficient |

---

## 1. List Comprehension

Creates the entire list in memory immediately.

### Syntax

```python
[expression for item in iterable if condition]
```

### Example

```python
numbers = [1, 2, 3, 4, 5]

squares = [x ** 2 for x in numbers]

print(squares)
print(type(squares))
```

**Output**

```text
[1, 4, 9, 16, 25]
<class 'list'>
```

**How it works:**

* Computes all values immediately.
* Stores them in a list.
* Easy to access any element by index.

---

## 2. Generator Expression

Generates values one at a time as needed.

### Syntax

```python
(expression for item in iterable if condition)
```

### Example

```python
numbers = [1, 2, 3, 4, 5]

squares = (x ** 2 for x in numbers)

print(squares)
print(type(squares))
```

**Output**

```text
<generator object ...>
<class 'generator'>
```

Retrieve values using a loop:

```python
for value in squares:
    print(value)
```

**Output**

```text
1
4
9
16
25
```

# Memory Usage Comparison

### List Comprehension

```python
numbers = [x for x in range(1000000)]

print(len(numbers))
```

* Creates **1,000,000** integers in memory.
* Requires significant RAM.

---

### Generator Expression

```python
numbers = (x for x in range(1000000))

print(numbers)
```

**Output**

```text
<generator object ...>
```

* Does **not** create all values at once.
* Produces each value only when requested.
* Uses much less memory.

---

# Accessing Elements

### List

```python
numbers = [x * 2 for x in range(5)]

print(numbers[2])
```

**Output**

```text
4
```

---

### Generator

```python
numbers = (x * 2 for x in range(5))

print(next(numbers))
print(next(numbers))
print(next(numbers))
```

**Output**

```text
0
2
4
```

The generator remembers its position and continues from where it left off.

---

# Generator is Consumed After Iteration

```python
gen = (x for x in range(5))

for i in gen:
    print(i)

print("Again:")

for i in gen:
    print(i)
```

**Output**

```text
0
1
2
3
4
Again:
```

The second loop prints nothing because the generator has already been exhausted.

# Performance Example

### List Comprehension

```python
numbers = [x ** 2 for x in range(10)]

print(sum(numbers))
```

---

### Generator Expression

```python
numbers = (x ** 2 for x in range(10))

print(sum(numbers))
```

Both produce:

```text
285
```

The generator avoids storing the intermediate list, making it more memory-efficient.

---

# Practical Example

Suppose you want the sum of squares of numbers from 1 to 1,000,000.

### Using List Comprehension

```python
total = sum([x ** 2 for x in range(1, 1000001)])

print(total)
```

* Creates a list of one million squared values before summing.

---

### Using Generator Expression

```python
total = sum(x ** 2 for x in range(1, 1000001))

print(total)
```

* Squares each number as `sum()` requests it.
* No large list is stored in memory.

# When to Use Which?

### Use List Comprehension When:

* You need all values at once.
* You need indexing or slicing.
* The dataset is relatively small.
* You will reuse the sequence multiple times.

Example:

```python
names = ["Alice", "Bob", "Charlie"]

upper = [name.upper() for name in names]

print(upper)
```

---

### Use Generator Expression When:

* Working with very large datasets.
* Reading large files line by line.
* Processing streaming data.
* Memory efficiency is important.
* Values are consumed only once.

Example:

```python
with open("data.txt") as file:
    lines = (line.strip() for line in file)

    for line in lines:
        print(line)
```

# Key Differences at a Glance

| Aspect              | List Comprehension | Generator Expression       |
| ------------------- | ------------------ | -------------------------- |
| Brackets            | `[]`               | `()`                       |
| Output              | List               | Generator                  |
| Memory              | High               | Low                        |
| Evaluation          | Immediate          | Lazy                       |
| Indexing            | Yes                | No                         |
| Multiple Iterations | Yes                | No (unless recreated)      |
| Suitable For        | Small datasets     | Large datasets and streams |

## Summary

* **List Comprehension (`[]`)**: Creates and stores the entire list immediately. Best when you need random access or will reuse the data.
* **Generator Expression (`()`)**: Produces values one at a time on demand. Best for large datasets, streaming data, and memory-efficient processing.
